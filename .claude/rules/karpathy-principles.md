# Karpathy Coding Principles

1. **Think Before Coding** — Clarify requirements, identify edge cases, never silently assume
2. **Simplicity First** — Minimal code, no unnecessary abstractions, if 3 lines work don't write 30
3. **Surgical Changes** — Touch only necessary code, don't refactor adjacent code
4. **Goal-Driven Execution** — Define verifiable success criteria, loop until met
