How TARS Tokens Are Used

Every trading decision flows through this routing pipeline:

1 Trading system submits a task (e.g., pre_trade_research)
2 ModelSelector maps task → complexity tier → canonical model ID
3 to_tars_model_id() translates to TARS-compatible provider string
4 OpenAI-compatible client sends request via LLM_GATEWAY_BASE_URL
5 TARS routes to optimal provider (DeepInfra, OpenAI, etc.)
6 Response returned; tokens + cost logged by TokenUsageMonitor
7 On failure → automatic fallback to OpenRouter direct (resilience)
