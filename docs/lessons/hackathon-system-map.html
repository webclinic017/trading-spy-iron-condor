How Our AI Trading System Works

One line summary: we run a continuous build-test-learn loop that improves trade logic, proves reliability with artifacts, and prepares a judge-ready demo automatically.

North Star: Better trades + proof Method: Layered TDD Router: Tetrate Memory: RAG synthesis

Explain Like I Am 10

  • We give the AI a checklist of small jobs.
  • It does one job, then checks if it works.
  • If it fails, it fixes it and tries again.
  • When it passes, it writes down what it learned.
  • Then it picks the next job and repeats.

Explain Like I Am an Engineer

  • `layered_tdd_loop.sh`: analyze -> implement -> verify.
  • `tars_autopilot.sh`: gateway, resilience, retrieval checks.
  • `devloop_commit_and_report.sh`: snapshots + PR check enforcement.
  • Artifacts feed KPI pages and RAG ingestion for next-cycle context.

Continuous Improvement Loop

1. Analyze tasks
->
2. Build minimal fix
->
3. Run tests/lint/smoke
4. Generate artifacts
->
5. Update RAG memory
->
6. Commit only when checks pass

System Blocks Working Together

  • Task Layer: Prioritized backlog in manual layer files.
  • Execution Layer: Codex/Copilot write and patch code.
  • Validation Layer: Unit tests, lint, smoke, resilience.
  • Evidence Layer: Scorecards, KPI, and demo artifacts.
  • Learning Layer: New outputs ingested into RAG for future runs.

Demo Status Model
