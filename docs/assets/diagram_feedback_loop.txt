Feedback-Driven Context Pipeline: Continuous Learning Architecture

This system implements a multi-stage feedback loop that captures user signals, extracts structured lessons, and injects relevant context into future sessions.

Stage 1 - Signal Capture:
User feedback (thumbs up, thumbs down, text corrections, frustration signals) is detected by hooks that run on every user prompt submission. Signals are classified by intensity (1-5) and category (code_edit, git, testing, pr_review, search, architecture, security, debugging).

Stage 2 - Thompson Sampling Model:
A Beta-Bernoulli model maintains reliability scores per task category. Each category has alpha (successes) and beta (failures) parameters. Exponential decay with 30-day half-life ensures recent feedback is weighted more heavily. Floor at 1% prevents complete zero-weighting of old feedback.

Stage 3 - Memory Storage (4 stores):
(a) Feedback Log (JSONL): raw append-only event log.
(b) LanceDB Vector Store: semantic embeddings of feedback for similarity search.
(c) MemAlign Episodes + Principles: full history distilled into reusable rules.
(d) ShieldCortex: persistent dual-memory store across sessions.

Stage 4 - Context Injection:
On session start, hooks query the Thompson model for weak categories (below 70% success), retrieve recent negative feedback from LanceDB via semantic search, and inject both into the system prompt. This makes the agent more cautious in historically weak areas.

The result: the system doesn't change model weights, but achieves behavioral adaptation through structured context engineering — the agent sees its past mistakes before acting.
