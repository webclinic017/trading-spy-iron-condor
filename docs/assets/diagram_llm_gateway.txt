System Architecture: Multi-Model LLM Gateway with TARS

The system routes all LLM inference calls through a unified gateway layer that supports three providers in a priority-based fallback chain.

Layer 1 - TARS Gateway (Tetrate Agent Router Service):
Primary provider. Routes to cost-optimized models: DeepSeek V3 for simple tasks ($0.30/1M tokens), Mistral Medium 3 for medium tasks ($0.40/1M tokens), DeepSeek-R1 for chain-of-thought reasoning ($0.70/1M tokens), and Kimi K2 for complex trading analysis ($0.39/1M tokens, ranked #1 on StockBench).

Layer 2 - OpenRouter (fallback):
If TARS is unavailable, routes through OpenRouter to the same model set.

Layer 3 - Anthropic Direct (critical only):
Trade execution and order placement always use Claude Opus ($15/1M tokens) directly via Anthropic API. This is never cost-cut regardless of budget.

Budget-Aware Model Selection (BATS Framework):
A ModelSelector class tracks daily spend ($0.83/day budget) and monthly spend ($25/month budget). Task complexity is classified as SIMPLE, MEDIUM, COMPLEX, or CRITICAL. The selector routes to the cheapest model that can handle the complexity, with automatic downgrade when budget is exceeded. Critical tasks bypass budget enforcement entirely.

The gateway resolves configuration from environment variables: LLM_GATEWAY_BASE_URL for the TARS endpoint, LLM_GATEWAY_API_KEY or TETRATE_API_KEY for authentication, and OPENROUTER_API_KEY for the fallback provider.
